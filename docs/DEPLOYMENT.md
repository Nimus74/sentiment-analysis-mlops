# Deployment

This document describes the deployment options available for the sentiment analysis system.

The project supports both **local development deployment** and **containerized deployment using Docker**, enabling reproducible environments and easier distribution of the application.

---

## Deployment Overview

The system can be deployed using the following approaches:

- Local Python environment
- Docker container
- Docker Compose environment

The main component exposed during deployment is the **FastAPI inference service**, which provides an HTTP API for sentiment prediction.

---

## System Components

During deployment the system may include the following components:

- **FastAPI Service** — provides model inference through REST API
- **Transformer Model** — primary sentiment classification model
- **FastText Model** — lightweight baseline model
- **Monitoring Dashboard (optional)** — Streamlit dashboard for monitoring reports

---

## Local Deployment

### 1. Clone the repository

```bash
git clone https://github.com/Nimus74/sentiment-analysis-mlops.git
cd sentiment-analysis-mlops
```

### 2. Create a virtual environment

```bash
python -m venv venv
source venv/bin/activate
```

On Windows:

```bash
venv\Scripts\activate
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
pip install -e .
```

---

## Running the API

Once dependencies are installed, the FastAPI server can be started with:

```bash
uvicorn src.api.main:app --host 0.0.0.0 --port 8000
```

The API will be available at:

```
http://localhost:8000
```

Interactive documentation is automatically generated by FastAPI and available at:

```
http://localhost:8000/docs
```

---

## Example API Request

Example request using Python:

```python
import requests

response = requests.post(
    "http://localhost:8000/predict",
    json={
        "text": "This product is amazing!",
        "model_type": "transformer"
    }
)

print(response.json())
```

The API returns the predicted sentiment and model confidence.

## Docker Deployment

The project includes a Dockerfile for containerized deployment.

### Build the Docker image

```bash
docker build -t sentiment-analysis-mlops .
```

### Run the container

```bash
docker run -p 8000:8000 sentiment-analysis-mlops
```

The API will then be accessible at:

```
http://localhost:8000
```

---

## Docker Compose Deployment

Docker Compose can be used to start multiple components together.

```bash
docker-compose up
```

This allows running:

- API service
- supporting services if configured
- monitoring components

in a single reproducible environment.

---

## Monitoring Dashboard (Optional)

If monitoring reports are generated, they can be visualized using the Streamlit dashboard.

Start the dashboard with:

```bash
streamlit run src/monitoring/dashboard.py
```

The dashboard will be available at:

```
http://localhost:8501
```
